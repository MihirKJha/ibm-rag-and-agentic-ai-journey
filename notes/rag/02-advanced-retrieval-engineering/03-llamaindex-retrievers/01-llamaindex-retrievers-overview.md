# 01. LlamaIndex Retrievers Overview

> **Category:** LlamaIndex Retrievers
> **Module:** Advanced Retrieval Engineering
> **Prerequisites:** RAG Fundamentals, Vector Databases, Embeddings, Retriever Fundamentals
> **Difficulty:** Intermediate

> **Note:** LlamaIndex provides a rich retrieval framework built around indexes, retrievers, query engines, node parsers, and postprocessors. Unlike traditional retrieval systems that focus only on vector similarity search, LlamaIndex offers multiple retrieval strategies optimized for different document structures, query types, and enterprise AI applications. This chapter provides an overview of the LlamaIndex retrieval architecture and introduces the major retrievers covered in this module.

---

# Overview

Retrieval is one of the most important components of a Retrieval-Augmented Generation (RAG) system. The quality of retrieved information directly impacts the quality of responses generated by a Large Language Model (LLM).

LlamaIndex extends traditional vector retrieval by providing a flexible retrieval architecture built around **Indexes**, **Retrievers**, **Query Engines**, and **Postprocessors**. These components work together to retrieve, rank, refine, and deliver relevant information to the LLM.

Instead of offering a single retrieval strategy, LlamaIndex provides multiple specialized retrievers designed for different enterprise use cases, such as keyword search, hierarchical documents, recursive retrieval, query fusion, and document summarization.

This modular architecture enables developers to build scalable, production-ready Retrieval-Augmented Generation (RAG) systems capable of handling diverse data sources and complex search requirements.

# Core Components

## Index

Indexes organize documents into structures that support efficient retrieval.

Examples include:

- Vector Store Index
- Summary Index
- Tree Index
- Keyword Table Index

Indexes determine how information is stored and searched.

---

## Retriever

Retrievers locate the most relevant nodes from an index.

Different retrievers use different retrieval strategies depending on the type of index and the search requirements.

---

## Node

A node represents the fundamental retrieval unit within LlamaIndex.

A node may contain:

- Text
- Metadata
- Relationships
- Embeddings
- References to parent nodes

Unlike simple document chunks, nodes preserve structural relationships between pieces of information.

---

## Query Engine

The Query Engine coordinates retrieval and response generation.

Responsibilities include:

- Executing retrieval
- Applying postprocessors
- Building prompts
- Invoking the LLM

---

## Node Postprocessor

Postprocessors improve retrieval quality after documents are retrieved.

Examples include:

- Similarity filtering
- Metadata filtering
- Sentence optimization
- Reranking

# Available Retrievers

This module covers the following LlamaIndex retrievers.

## Vector Index Retriever

Retrieves nodes using vector similarity search.

Best for

- Semantic search
- General-purpose RAG

---

## BM25 Retriever

Performs keyword-based retrieval using the BM25 ranking algorithm.

Best for

- Exact keyword matching
- Technical documentation
- Product codes

---

## Document Summary Retriever

Retrieves document summaries before accessing detailed content.

Best for

- Long documents
- Research papers
- Enterprise documentation

---

## Recursive Retriever

Traverses relationships between nodes to retrieve connected information.

Best for

- Hierarchical knowledge
- Multi-level documents
- Linked content

---

## Query Fusion Retriever

Executes multiple query variations and merges the results into a unified ranking.

Best for

- Ambiguous questions
- Improved recall
- Enterprise search

---

## Auto Merging Retriever

Retrieves fine-grained nodes and automatically merges related nodes into larger contextual units.

Best for

- Long documents
- Technical manuals
- Context preservation

# When to Use Each Retriever

| Retriever | Best For |
|------------|----------|
| Vector Index Retriever | Semantic Search |
| BM25 Retriever | Keyword Search |
| Document Summary Retriever | Long Documents |
| Recursive Retriever | Hierarchical Knowledge |
| Query Fusion Retriever | Complex Queries |
| Auto Merging Retriever | Context Preservation |

# Enterprise Use Cases

### Enterprise Knowledge Bases

Use Vector Index Retriever for semantic search across documentation.

---

### Technical Documentation

Combine BM25 Retriever with Vector Index Retriever for hybrid retrieval.

---

### Legal Search

Use Recursive Retriever to traverse linked legal documents.

---

### Research Platforms

Use Document Summary Retriever to quickly identify relevant reports before retrieving detailed content.

---

### Enterprise AI Assistants

Combine Query Fusion Retriever with Auto Merging Retriever to improve recall while preserving document context.

# Key Takeaways

- Node relationships enable advanced retrieval strategies that go beyond traditional vector search.
- Understanding the overall retrieval architecture is essential before exploring individual LlamaIndex retrievers.
